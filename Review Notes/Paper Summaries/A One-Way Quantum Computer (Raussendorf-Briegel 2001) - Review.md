# Review: A One-Way Quantum Computer (Raussendorf-Briegel 2001)

Source note: [[A One-Way Quantum Computer (Raussendorf-Briegel 2001) — Paper Notes]]

## Primary Sources Checked

- Raussendorf and Briegel, "A one-way quantum computer," Phys. Rev. Lett. 86, 5188-5191 (2001); arXiv:quant-ph/0010033.

## Verdict

Minor-to-major issues. The conceptual summary is strong, but two pedagogical shortcuts are too aggressive: the CNOT pattern is understated, and the "no pre-written input required" statement needs to distinguish state preparation from processing an arbitrary unknown quantum input.

## Line-Anchored Comments

- Lines 7-12: excellent high-level description. The "same state, different measurement pattern" framing is the right intuition for MBQC.
- Lines 17-25: the cluster-state construction and stabilizers are correct. The stabilizer eigenvalue sign depends on preparation conventions and any prior `Z` measurements, so avoid treating the displayed `+/-` as a fixed defining equation without specifying the convention.
- Lines 31-35: the wire-as-teleportation explanation is good. Mention that the wire usually implements Hadamards interleaved with byproducts; the logical identity wire depends on chain length and basis choices.
- Lines 39-45: good account of adaptive one-qubit rotations. This is where non-Clifford measurement angles enter and where universality escapes stabilizer simulability.
- Lines 46-48: likely too compressed. The original one-way CNOT construction is not just a four-qubit cross in the usual resource accounting; standard presentations use a larger cluster pattern with measured internal sites and byproduct tracking. Recheck the exact Raussendorf-Briegel pattern before quoting "requires 4 qubits."
- Lines 67-77: byproduct propagation is a strength of the note. The sign-flip/adaptivity connection is exactly the mechanism that makes the model deterministic.
- Lines 81-83: needs qualification. Measurements on a fixed cluster can prepare known input states, but an arbitrary unknown quantum input must be injected/attached to the cluster. "No pre-written input required" should not read as if unknown input data can be generated by measurement alone.
- Lines 87-94: physical implementation section is useful historically, but the percolation/filling numbers are not central to the theorem and should be dated as 2001-era implementation context.
- Lines 112-117: caveats are good. Add that fault-tolerant MBQC is mostly understood through later topological/foliated cluster-state constructions, not this original PRL alone.

## Missing or Suggested Additions

- Add a tiny diagram or table of measurement bases for wire, one-qubit rotations, and CNOT with the actual cluster sizes from the paper.
- Add a "known input preparation vs unknown input injection" distinction.
